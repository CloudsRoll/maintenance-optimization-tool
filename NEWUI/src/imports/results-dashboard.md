3. Results Dashboard

This is one of the most important pages.

Purpose

Display the results of the optimization model in a clear and understandable way.

Content
Summary Section

Show key results in summary cards.

Example cards:

Optimal Expected Cost

Number of Components

Model Type

Key Recommendation

Results Table

Display the optimal actions for different system states.

Example columns:

Environment State

Deterioration Level

Recommended Action

Intervention Status

Visualizations

Include simple charts to help users understand the results.

Examples:

Cost comparison chart

Maintenance policy chart

Action by deterioration level

Interpretation Section

Provide a short explanation of the results.

Example:
“The optimal policy recommends preventive maintenance once the system reaches deterioration level 3 in environment state 1.”

4. Model Explanation Page (Method / Theory)
Purpose

Explain the model for users who want to understand how it works.

Content

Short explanation of the Markov Decision Process model

Description of deterioration states

Description of maintenance actions

Explanation of how the optimization works

Diagram of the system or state transitions

This page helps users trust the results and understand the methodology.

5. How to Use Page (User Guide)
Purpose

Explain how users should interact with the tool.

Content

Step-by-step instructions such as:

Enter system parameters in the input page.

Click “Run Optimization”.

Wait for the model to compute the results.

Review the optimal maintenance policy in the results dashboard.

Use the charts and tables to analyze the system behavior.

This page is especially helpful for new users.

6. About the Project Page
Purpose

Explain the background of the project.

Content

Short project description

University / course information

Team members

Supervisor

Project objective

Example sections:

Project Background

Research Motivation

Team Members

7. Optional: Scenario Comparison Page

If your model later supports running multiple parameter sets, you can add a comparison page.

Purpose

Allow users to compare different scenarios.

Content

Comparison of optimal costs

Comparison of maintenance policies

Charts showing differences between scenarios

This is optional but useful if the project expands.

Home

Summary

Your web application should include the following pages:

Home Page – introduction and entry point

Model Setup Page – parameter input

Results Dashboard – optimization results and visualizations

Model Explanation Page – explanation of the MDP model

How to Use Page – user instructions

About Page – project information

This structure ensures that users can understand the tool, run the optimization, and interpret the results easily, even if they do not have technical knowledge of the underlying model.
